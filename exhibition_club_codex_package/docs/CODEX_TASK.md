# Codex 작업 지시서

## 목표

전시 및 공연 동호회를 위한 모바일 안내 페이지와 Google 설문 운영 시스템을 완성한다.

구성은 다음과 같다.

- 모바일 안내 페이지 1개
- Google Form 2개
  - 7월 정기 관람 참석 설문
  - 전시·공연 추천 설문
- Google Sheet 1개
  - 회원/참석/추천 전시 운영 관리

## 현재 동호회 운영안

- 동호회 성격: 함께 보고, 즐기고, 나누는 전시·공연 동호회
- 7월 정기 관람: 퐁피두센터 한화 63빌딩 《큐비스트》
- 주말 관람: 2026.7.11.(토) 16:00 / 28,000원
- 평일 관람: 2026.7.29.(수) 19:00 / 14,000원 / 문화의날 50% 할인
- 자율 관람: 국립현대미술관, 서울시립미술관 등 희망자끼리 자유롭게 진행
- 매월 운영 흐름:
  - 첫째 주: 보고 싶은 전시/공연 제안
  - 둘째 주: 후보 투표
  - 셋째 주: 관람 일시 투표
  - 넷째 주: 확정 안내

## 작업 범위

### 1. 모바일 안내 페이지

`public/index.html`, `public/styles.css`, `public/config.js`를 점검하고 다음 조건을 만족하도록 수정한다.

- 모바일 우선 디자인
- 긴 설명보다 버튼 중심
- 주말 관람/평일 관람 정보가 명확히 보일 것
- 참석 설문 버튼과 전시 추천 버튼이 눈에 잘 보일 것
- 참고 링크는 카드 형태로 제공
- 개인정보 안내 문구 포함
- Form 링크는 `public/config.js`에서만 수정하도록 유지

### 2. Google Apps Script

`apps-script/Code.gs`를 점검하고 다음이 가능하게 한다.

- Google Sheet 1개 생성
- Google Form 2개 생성
- 두 Form 응답을 같은 Google Sheet에 연결
- 설정 시트에 Form 편집 URL, 응답 URL, Sheet URL 기록
- 운영진이 볼 수 있는 기본 시트 생성

### 3. 배포 문서

README에 다음 내용을 보완한다.

- GitHub Pages 배포 방법
- Vercel 배포 방법
- Google Apps Script 실행 방법
- Form 링크를 config.js에 넣는 방법
- 카카오톡 단톡방 공유 방법

## 완료 기준

- `public/index.html`을 로컬에서 열면 모바일 안내 페이지가 정상 표시된다.
- `public/config.js`에서 Form URL만 바꾸면 버튼 링크가 변경된다.
- `apps-script/Code.gs`를 실행하면 Google Form 2개와 Google Sheet 1개가 생성된다.
- 운영진이 README만 보고 따라 할 수 있다.

## 주의사항

- 개인정보는 최소한만 수집한다.
- 전화번호는 처음 버전에서 수집하지 않는다.
- 결제 기능은 만들지 않는다.
- 로그인 기능은 만들지 않는다.
- 현재는 v0.1이므로 지나치게 복잡한 기능을 넣지 않는다.
